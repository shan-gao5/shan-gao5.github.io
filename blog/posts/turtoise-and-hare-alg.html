##mcdouble algorithm This came up too many times for no note on it

Suppose there is an directed graph where each node as one directed neighbour (like a linked list) on which there is a cycle and we want to find the starting point of the cycle.

Let $f = s$ be the head of the linked list, corresponding to the fast and slow pointers

Iterate as such:

while f != s:
	f= f.next.next
	s = s.next

When this halts, $f$ and $g$ will meet in the cycle.

Let $a=$ distance from head until start of cycle $b=$ distance from the start of cycle to the slow pointer $c=$ remaining distance from the slow pointer to the start of the cycle

First of all, it's obvious that $s$ will enter the cycle if there exists one, after $f$ enters it.

letting $C$ be the length of the cycle, $|f - s| = d$ for some $d >0$ then increases by $1$ every iteration $d$, $d + 1$, $d + 2,\ldots$ until $|f - s| = 0 \mod C$, that's when they meet. They will meet before slow completes a full revolution.

Let $a=$ distance(head, start of cycle) $b=$ distance(start of cycle, slow pointer) $c =$ remaining distance from slow pointer to start of cycle

$s$ has travelled distance $a + b$, and $f$ has travelled distance $2(a+b)$, the difference between their distance travelled has to be a multiple of the cycle length (why?) thus

$a + b = k ( b + c)$ rearranging $$ a = (k-1)b + kc = (k-1)b + (k-1)c + c = (k-1)(b+c) + c $$ Thus $$ a = c \mod{C} $$
